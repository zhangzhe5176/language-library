## 4. 重复词与状态规则

稳定词卡记录 ID 为：

```text
{level}:t{topicId}:s{storyId}:v{sourceVocabNo}
```

该 ID 不依赖当前显示顺序；9,116 条原始记录无 ID 冲突。

学习状态与词卡记录分开保存。状态键为：

```text
{level}:word:{NFKC 规范化的原始词形}
```

同一等级中相同规范化词形跨 Topic 或在同一 Topic 内重复出现时，共用一份状态；但每一条原始记录仍保留独立词卡。修改其中任意一张卡会同步其他相同词卡。Topic 统计按原始词卡记录计数，因此重复词的一个状态可能同时计入多张卡；这是产品要求“保留全部记录”和“同词共享状态”共同产生的统计语义。

重复记录统计：同一 Topic 内重复额外记录 115 条；跨 Topic 共有 45 个规范化词形、94 条原始记录，除每个词形首次记录外的跨 Topic 额外记录为 49 条；总重复额外记录为 164 条。

## 5. 本地状态迁移

旧版本存储键：`languageLibrary:vocabularyPrototype:n5:v1`

候选版存储键：`languageLibrary:vocabularyPrototype:all:v2`

启动时仅在新版本存储不存在时读取旧 N5 状态，通过旧行 ID 映射到新的共享词形状态键；旧记录不会被删除，无法解析时也不会覆盖或清空旧值。N5～N1 的状态按等级隔离，当前等级和各等级最近访问 Topic 同时保存在 v2 状态中。

## 6. 结论

项目现有数据可以安全加载 N5～N1 全部 105 个 Topic 和全部 9,116 条目标生词记录。当前原型不再通过去重减少词卡；20 条无法在项目内确认读音的记录被保留在原始位置并进入待核对状态，未用汉字、中文释义或语言模型知识替代读音。
