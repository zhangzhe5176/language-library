## 4. 重复词与唯一 ID 规则

稳定行 ID 为：

```text
{level}:t{topicId}:s{storyId}:v{sourceVocabNo}
```

该 ID 不依赖当前显示顺序；五个等级共 9,116 个原始词条没有 ID 冲突。

学习状态使用等级范围内的稳定词形键：

```text
{level}:word:{NFKC 规范化的原始词形}|{NFKC 规范化的假名}
```

同一等级、同一词形在多个 Topic 出现时共用一份状态；同一 Topic 内重复数据只保留原始顺序中的首次记录进入词卡、统计和导出。这样避免重复卡片，同时保持 Topic 与目标生词关系。共享词形的状态遵循“同一等级全站一份状态”规则。

## 5. N5 旧原型状态迁移

旧版本存储键：`languageLibrary:vocabularyPrototype:n5:v1`

候选版存储键：`languageLibrary:vocabularyPrototype:all:v2`

启动时仅在新版本存储不存在时读取旧 N5 状态，通过旧行 ID 映射到新的稳定词形键；旧记录不会被删除，无法解析时也不会覆盖或清空旧值。验证结果：旧 N5 状态可迁移，旧键仍保留，N4～N1 从未筛查开始。

## 6. 结论

N5～N1 均有完整的 Topic 关系、词性和中文释义，原型可以安全加载五个等级的全部 Topic 和去重后的全部目标生词。数据中缺少假名和汉字的数量已如实保留并在页面侧采用保守兼容；是否需要后续人工补齐，留待用户完整验收决定。
