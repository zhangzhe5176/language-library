## 12. 正式发布步骤

1. 提交并推送 `feature/audio-compression`。
2. 确认 feature 分支工作区干净、原始音频基线一致、报告结论为 Go。
3. 只用压缩后的 `dist/` 内容更新 `gh-pages` 根目录。
4. 提交：`deploy: publish compressed audio`。
5. 正常推送 `origin/gh-pages`，不 orphan、不强制推送、不改写历史。
6. 等待 GitHub Pages 部署成功。
7. 在线检查根门户、N1–N5 各至少 5 条音频，覆盖前/中/后，并检查播放进度、404 和控制台。
8. 线上通过后，`main` 使用 `git merge --ff-only feature/audio-compression`，再提交发布记录。

## 13. 回滚方案

- 线上出现 Critical 问题时立即停止，不合并 `main`。
- 在 `gh-pages` 上使用上一个发布提交创建正常回退提交并推送，不强制推送。
- 保留 feature 分支和本地缓存，便于复现和修复。
- 源码原始音频不受发布回滚影响。
